# 20260419 工作流改进 - 远程同步模块化与缓存结构优化

## 📋 改进概览

本次改进围绕三个核心目标：
1. **模块化远程同步脚本** - 将远程同步逻辑从测试文件夹提升到专用模块
2. **标准化缓存结构** - 统一 summary 缓存格式，按类型分文件夹存储（对齐 OSS 结构）
3. **灵活的传输控制** - 保留和优化开关参数，支持灵活的本地/远程模式

## 🏗️ 新增：`backend/remote_sync/` 模块

### 目录结构
```
backend/remote_sync/
```

### 主要文件：`run_remote_sync_smoke.py`

**功能**：提供可复用的端到端远程同步测试，验证 Supabase 和 Aliyun OSS 连接。

**参数**：
```python
run_remote_sync_smoke_test(
    user_id: str = "u_demo_001",              # 用户标识
    book_id: str = "b_demo_001",              # 书籍标识
    source_file_id: str = "sf_test_fullchain_002",  # 源文件标识
    cache_base_dir: str | Path = None,        # 缓存根目录（可选）
    skip_remote: bool = False,                # 跳过远程操作，仅验证本地缓存
) -> dict
```

**使用方式**：

```bash
# 标准远程同步测试
cd e:\codes\CharPick
conda activate charpick
python -m backend.remote_sync.run_remote_sync_smoke

# 跳过远程操作（本地验证）
SKIP_REMOTE=true python -m backend.remote_sync.run_remote_sync_smoke

# 自定义参数（通过环境变量）
CUSTOM_USER_ID=u_demo_002 CUSTOM_BOOK_ID=b_demo_002 python -m backend.remote_sync.run_remote_sync_smoke
```

**返回值**：
```json
{
  "chapter_task_id": 1,
  "summary_task_id": 2,
  "card_task_id": 3,
  "l0_l1": { "uploaded_chapter_count": ... },
  "l2": { "uploaded_extraction_count": ... },
  "summary": { "uploaded_summary_count": ... },
  "card": { "uploaded_card_count": ... }
}
```

### 旧结构 ❌
```

├── summary_json/
│   ├── {source_file_id}_characters.json
│   ├── {source_file_id}_items.json
```

### 优势
- ✅ **对齐 OSS 结构** - 本地和远程存储格式一致
- ✅ **易于导航** - 按维度分类，更直观
- ✅ **扩展性** - 新增维度时无需修改代码
- ✅ **一致性** - Card 缓存也采用类似结构

## 🔧 修改详情

### 1. `backend/llm_dispatcher.py` - `dispatch_summary_prompts()`

**改动**：修改缓存目录创建逻辑

```python
# 旧代码
summary_json_dir = summary_root_dir / "summary_json"
summary_md_dir = summary_root_dir / "summary_md"
summary_json_dir.mkdir(parents=True, exist_ok=True)
summary_md_dir.mkdir(parents=True, exist_ok=True)
output_json_path = summary_json_dir / f"{source_file_id}_{dimension}.json"
output_md_path = summary_md_dir / f"{source_file_id}_{dimension}.md"

# 新代码
dimension_dir = summary_root_dir / dimension  # 按维度创建文件夹
dimension_dir.mkdir(parents=True, exist_ok=True)
output_json_path = dimension_dir / f"{dimension}.json"
output_md_path = dimension_dir / f"{dimension}.md"
```

**返回值调整**：
```python
# 移除 summary_json_dir 和 summary_md_dir
return {
    "summary_rows": summary_rows,
    "summary_root_dir": summary_root_dir.as_posix(),
    "llm": llm_cfg,
}
```

### 2. `backend/remote_persistence.py` - `persist_summary_rows()`

**改动**：适配新的本地路径结构

```python
# 旧代码
local_folder = json_path.parent.parent.as_posix()  # 指向 summary/ 文件夹

# 新代码
local_folder = json_path.parent.as_posix()  # 指向 summary/{dimension}/ 文件夹
```

### 3. `backend/test/test_llm_dispatcher.py`

**改动**：更新测试断言以验证新的文件结构

```python
# 旧代码
assert Path(result["summary_json_dir"]).exists()
assert Path(result["summary_md_dir"]).exists()
assert (chapter_dir.parent / "summary" / "summary_json" / "sf_test_001_characters.json").exists()
assert (chapter_dir.parent / "summary" / "summary_md" / "sf_test_001_characters.md").exists()

# 新代码
assert Path(result["summary_root_dir"]).exists()

assert (chapter_dir.parent / "summary" / "characters" / "characters.md").exists()
```

## 🎛️ 远程传输控制开关

### 在 `run_l0_to_l2_pipeline()` 中的开关参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `upload_chapters_to_remote` | `False` | 是否上传 L0/L1/L2 数据到 Supabase 和 OSS |
| `upload_summary_to_remote` | `True` | 是否上传 summary 数据 |
| `upload_card_to_remote` | `True` | 是否上传 card 数据 |
| `run_summary` | `True` | 是否运行 summary 任务 |
| `run_card` | `True` | 是否运行 card 任务 |

### 使用示例

```python
from backend.workflow_runner import run_l0_to_l2_pipeline

# 仅本地处理（不上传到远程）
result = run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=False,
    upload_summary_to_remote=False,
    upload_card_to_remote=False,
)

# 仅生成 summary 和 card，不上传 chapter 数据
result = run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=False,
    upload_summary_to_remote=True,
    upload_card_to_remote=True,
    run_summary=True,
    run_card=True,
)

# 完整管道：所有数据都处理和上传
result = run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=True,
    upload_summary_to_remote=True,
    upload_card_to_remote=True,
    run_summary=True,
    run_card=True,
)
```
