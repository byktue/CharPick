#### 2️⃣ 统一 Summary 缓存结构

**旧结构 (分离)：**
```
summary/
├── summary_json/
│   ├── {id}_characters.json
│   ├── {id}_items.json
│   └── ...
└── summary_md/
    ├── {id}_characters.md
    ├── {id}_items.md
    └── ...
```

**新结构 (分类)：**
```
summary/
├── characters/
│   ├── characters.json
│   └── characters.md
├── items/
│   ├── items.json
│   └── items.md
├── storyline_events/
│   ├── storyline_events.json
│   └── storyline_events.md
└── world_locations/
    ├── world_locations.json
    └── world_locations.md
```

**优势：**
- ✅ 与 OSS 存储格式一致
- ✅ 按维度分类，更直观
- ✅ 易于扩展新维度
- ✅ 脚本支持向后兼容

#### 4️⃣ 远程传输控制开关

在 `run_l0_to_l2_pipeline()` 中已有以下控制开关：

```python
# 控制是否上传各类数据到远程
upload_chapters_to_remote: bool = False      # L0/L1/L2 数据
upload_summary_to_remote: bool = True        # Summary 数据
upload_card_to_remote: bool = True           # Card 数据

# 控制是否运行各类任务
run_summary: bool = True                     # 是否运行 summary 任务
run_card: bool = True                        # 是否运行 card 任务
```

**灵活的使用示例：**
```python
# 仅本地处理
run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=False,
    upload_summary_to_remote=False,
    upload_card_to_remote=False,
)

# 上传 summary 和 card 到远程，但不上传 chapter 数据
run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=False,
    upload_summary_to_remote=True,
    upload_card_to_remote=True,
)

# 完整管道
run_l0_to_l2_pipeline(
    source_path="data/novel.txt",
    upload_chapters_to_remote=True,
    upload_summary_to_remote=True,
    upload_card_to_remote=True,
)
```
