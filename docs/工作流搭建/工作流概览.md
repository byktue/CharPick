数据库：本地缓存（json） +  云端supabase + 云端oss存储大文件
工作流中llm调度版块选择：vllm，ollama，远程api
程序语言：python

输入 → 多源文件解析 → 文本清洗（统一成txt中间输出） → 章节 / 逻辑分块 → LLM 逐章结构化提取 → 提取summary信息 → 依据summary信息提取的角色总表信息，选择相应角色（通过“角色名”进行选择） → 检索逐章对应角色信息（当前还使用的是Json检索，还并未实装RAG检索功能） → 角色 Prompt 生成 → 前端展示角色库  → 复制粘贴角色库提供的prompt，调用api与llm进行交互（具有本地记忆缓存）

| 层级       | 对应业务表                                          | 这层存什么                                                     |
| -------- | ---------------------------------------------- | --------------------------------------------------------- |
| L0 原始层   | books、source_files                             | 书籍主入口、原始文件、导入元数据、OCR 来源                                   |
| L1 章节层   | chapters（本地缓存）                                 | 切分后的章节文本                                                  |
| L2 提取层   | chapter_extractions                            | 每章抽出的角色、物品、剧情、地点、时间线 JSONB，本地缓存，且传输到远程数据库（supabase + oss） |
| L3 汇总层   | summary                                        | 面向前端展示书籍汇总提取的结果（人物、物品、剧情、世界观）                             |
| L4 展示调度层 | card                                           | 面向前端展示角色库的最终业务结果                                          |
| 通用支撑     | users、auth_sessions、extraction_tasks、task_logs | 用户身份、登录态、任务过程、任务日志                                        |
