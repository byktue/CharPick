# CharPick 数据库表结构汇总（**带精简注释版**）
所有表统一放在 **单个 PostgreSQL 库**，通过 **user_id 做数据隔离**，直接对接前端接口。

| 层级     | 对应业务表                                                                             | 这层存什么                      |
| ------ | --------------------------------------------------------------------------------- | -------------------------- |
| L0 原始层 | books、source_files                                                                | 书籍主入口、原始文件、导入元数据、OCR 来源    |
| L1 章节层 | chapters                                                                          | 切分后的章节文本、章节编号、章节范围、章节定位    |
| L2 提取层 | chapter_extractions                                                               | 每章抽出的角色、物品、剧情、地点、时间线 JSONB |
| L3 汇总层 | characters、character_versions、items、storyline_events、world_locations、media_assets | 面向前端和角色库的最终业务结果            |
| 通用支撑   | users、auth_sessions、extraction_tasks、task_logs                                    | 用户身份、登录态、任务过程、任务日志         |


---
## 一、用户与认证表
### 1. users（用户主表）
- id（PK）# 用户唯一ID，主键 M
- username # 用户名 M
- email # 邮箱
- password_hash # 密码哈希（不存明文） M
- status # 账号状态：正常/禁用
- created_at # 创建时间 M
- updated_at # 更新时间 M

### 2. auth_sessions（会话/Token 表）
- id（PK）# 会话唯一ID M
- user_id（FK → users）# 所属用户（外键） M
- access_token_jti # 访问令牌唯一标识 M
- refresh_token_hash # 刷新令牌哈希 M
- expires_at # 过期时间 M
- revoked_at # 撤销时间（NULL=有效）
- created_at # 登录时间 M
- updated_at # 最后更新时间 M

---

## 二、书籍与原始文件表
### 3. books（书籍/项目主表）
- id（PK）# 书籍唯一ID M
- user_id（FK → users）# 所属用户（外键） M
- title # 书名/项目名 M
- author # 原著作者
- source_type # 来源类型：epub/txt/pdf/image M
- language # 语言：zh-CN/en
- status # 处理状态：待解析/解析中/已完成/失败 M
- cover_url # 封面图片地址
- created_at # 创建时间 M
- updated_at # 更新时间 M

### 4. source_files（源文件表 L0）
- id（PK）# 源文件ID M
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- file_name # 原始文件名 M
- file_url # 文件存储地址 M
- file_hash # 文件哈希（防重复） M
- mime_type # 文件类型
- unified_format_url # 中间结构地址（实际文件存储在本地缓存，本地缓存用txt格式存储）
- parse_status # 解析状态 M
- created_at # 创建时间 M
- updated_at # 更新时间 M

---

## 三、章节与提取表（L1/L2）
### 5. chapters（章节表 L1）（章节切分得到的结果）
- id（PK）# 章节ID M
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- chapter_no # 章节序号 M
- chapter_title # 章节标题
- chapter_range # 章节范围（精准标记这一章在**原始文件中的起止位置**）
- content_text_url # 章节纯文本地址（实际文件存储在本地缓存，本地缓存用txt格式存储）
- word_count # 字数统计
- source_file_id（FK → source_files）# 来源文件
- created_at # 创建时间 M
- updated_at # 更新时间 M
### 6. chapter_extractions（章节提取表 L2）
- id（PK）# 提取记录ID M
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- chapter_id（FK → chapters）# 所属章节 M
- extraction_json_url（TEXT）# 章节提取结构化数据 （oss存储）
- embedding_vector（pgvector）# 向量数据
- prompt_version # 提示词版本
- model_name # 使用模型
- confidence # 置信度
- created_at # 创建时间 M
- updated_at # 更新时间 M

补充extractor_type解释

| 常见 `extractor_type` 枚举值 | 对应提取任务                       |
| ----------------------- | ---------------------------- |
| `character`             | 角色提取（提取本章出现的角色、人设）           |
| `plot`                  | 剧情提取（提取本章的剧情事件、时间线）          |
| `item`                  | 物品提取（提取本章出现的道具、装备）           |
| `location`              | 地点提取（提取本章出现的场景、地点）           |
| `summary`               | 章节摘要提取（生成本章的总结）              |
| `full`                  | 全量提取（一次性提取角色 + 剧情 + 物品等所有内容） |
