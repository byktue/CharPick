### 角色数据库 与 详细的提取数据库分为两个
角色数据库用于存储可用的已经简化过的角色prompt
而详细的提取数据库则存储相应的提取全过程完整的数据

利用 **RAG (检索增强生成)** 技术，将长文本切片存储在向量数据库中。通过专门的 `PersonaExtractor` 链，识别文本中的“外貌”、“性格”、“核心事件”和“口癖”

|**维度**|**主流做法**|**难点**|
|---|---|---|
|**长文本处理**|递归总结 (Recursive Summarization) 或 向量检索 (RAG)|避免角色性格在长文本后期的“漂移”|
|**人设结构化**|提取为 JSON 格式（含：姓名、动机、禁忌语、背景）|捕捉角色细微的口癖（Linguistic Habits）|
|**多模态联动**|Text-to-Image (生成形象) + TTS (生成音色)|视觉形象与文本描述的高度统一|

# 相关技术建议

## 1. 开源界：成熟的框架与项目

### **ChatHaruhi (凉宫春日项目)**

这是目前国内在“长文本提取并模拟角色”领域最知名的开源项目之一。

- **核心逻辑**：它通过从原始剧本、小说中抽取大量对话片段，构建“角色记忆库”。当用户提问时，系统会搜索相关的经典情节，结合 Prompt 驱动 LLM 模仿。
    
- **多模态能力**：最新版本已经集成了多模态支持，可以根据角色性格生成表情包、动作描述甚至对应的语音（TTS）。
    
- **适用场景**：如果你想从已有的小说或剧本中快速复刻一个可以对话的角色，这是首选方案。
    

### **CharacterGLM (清华/聆心智能)**

虽然 CharacterGLM 更多作为商业模型提供，但其背后的技术架构（如 **CoAI** 课题组的研究）是行业标杆。

- **特点**：专门针对“人设一致性”进行微调，支持超长上下文（32k+ token）。
    
- **多模态表现**：它不仅能生成文本，还能在回复中自动加入表情符号、动作描写（如“_摸了摸头说_”），并与多模态生成模型联动。
    

### **LangChain / LlamaIndex 的 Agent 方案**

很多开发者基于这些通用框架自研了“角色提取器”：

- **做法**：利用 **RAG (检索增强生成)** 技术，将长文本切片存储在向量数据库中。通过专门的 `PersonaExtractor` 链，识别文本中的“外貌”、“性格”、“核心事件”和“口癖”。
    
- **多模态扩展**：通常外接 Stable Diffusion 或 Midjourney 的 API，根据提取出的视觉特征（Visual Description）生成角色立绘。

# 角色行为优化
1. **从“静态描述”到“动态行为”**：
           目前的趋势不再仅仅是提取“性格：高冷”，而是提取“行为逻辑”。例如： 这种架构可以将提取到的信息转化为角色的“记忆模块”。
    
2. **多模态融合的实现路径**：
    - **视觉**：利用提取到的 Character Persona 生成稳定的提示词（Prompts），通过 `controlnet` 保持角色立绘的一致性。
    - **音频**：利用 GPT-SoVITS 等项目，通过提取小说中对角色声音的描述来寻找或训练匹配的音色。
