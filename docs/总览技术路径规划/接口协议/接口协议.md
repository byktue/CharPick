# 📡 1. 上传与任务处理接口 (Task & Upload)
**功能**：处理文件上传、启动 L0-L1 解析，并提供进度查询。

### 文件上传
- **方法**：POST
- **路径**：`/api/v1/upload`
- **主要输入参数**：
  - `file`: Multipart
  - `user_id`: str
  - `book_name`: str（可选）
- **业务逻辑**：若 `book_name` 为空，后端自动抓取文件名作为书名存储。生成任务并返回 ID。

**输出示例（JSON）**
```json
{
  "task_id": "t_001",
  "book_id": "b_123",
  "status": "pending"
}
```

---

### 处理进度查询
- **方法**：GET
- **路径**：`/api/v1/task/{task_id}`
- **主要输入参数**：`task_id`（路径参数）
- **业务逻辑**：对应 L0 存储、L1 章节解析等阶段的实时进度可视化。

**输出示例（JSON）**
```json
{
  "progress": 65,
  "stage": "L1_Parsing",
  "status": "running"
}
```

---

### 手动触发提取
- **方法**：POST
- **路径**：`/api/v1/extract`
- **主要输入参数**：
  - `book_id`: str
  - `roles`: list
  - `is_dynamic`: bool
- **业务逻辑**：根据总表勾选角色进行深度提取。`is_dynamic` 开启则识别形象改变节点。

**输出示例（JSON）**
```json
{
  "task_id": "t_002",
  "message": "Extraction started"
}
```

# 👤 3. 角色库与动态形象接口 (Character Persona)
**功能**：核心业务逻辑，处理「全局形象」与「动态形象」的切换。

### 已生成角色库
- **方法**：GET
- **路径**：`/api/v1/characters`
- **说明**：获取当前 `user_id` 下所有书籍中已完成提取的角色卡快照

**输出示例（JSON）**
```json
{
  "total": 42,
  "characters": [
    {
      "char_id": "c_01",
      "name": "石野",
      "from_book": "神游",
      "book_id": "b_123",
      "avatar_url": "/assets/chars/c_01.png"
    }
  ]
}
```

---

### 角色详情（全局）
- **方法**：GET
- **路径**：`/api/v1/characters/{char_id}`
- **说明**：获取该角色的 L2 级全局静态画像（基础属性、性格、核心设定）
	number代表编号，0代表全局静态画像，其他编号依据chapter_range进行依次编号
	chapter_range代表符合动态画像的章节范围（如果number为0，则chapter_range也为0即可）

**输出示例（JSON）**
```json
{
  "char_id": "c_01",
  "number": "0",
  "chapter_range": "0",
  "detail": "【姓名】【性别】【年龄】【种族】【能力】【社会关系】等这些详细的角色信息"
}
```
