# CharPick 完整测试流程

本文档用于验证三件事：

1. 后端程序是否连通。
2. 前端程序是否连通。
3. 前后端是否真正联通，并能通过真实页面完成上传、提取、角色卡和聊天验证。

测试顺序建议固定为：

1. 先跑脚本级功能测试和连通测试。
2. 再启动后端服务与前端服务。
3. 最后打开真实前端页面执行端到端验证。

---

## 1. 测试前准备

先确认工作目录是仓库根目录：`E:\codes\CharPick`。

建议先切换到后端 Python 环境：

```powershell
conda activate charpick
```

前端工程位于：`frontend/frontend_vue`。

后端默认监听：`http://127.0.0.1:8000`

前端默认开发地址：`http://127.0.0.1:5173`

前端默认后端地址已经写在 `frontend/frontend_vue/src/stores/settings.js`，默认值是 `http://127.0.0.1:8000`。

---

## 2. 第一阶段：脚本级功能测试和连通测试

这一阶段不需要打开浏览器页面，先验证底层能力是否可用。

### 2.1 后端单元和回归测试

在仓库根目录运行：

```powershell
python -m pytest -q backend/test/test_llm_dispatcher.py backend/test/test_llm_remote_client.py
```

如果你想跑更完整的轻量测试集，可以运行：

```powershell
python test/run_full_tests.py
```

`test/run_full_tests.py` 默认会执行：

- `test/backend_tests.py`
- `test/frontend_simulation.py`
- `test/db_tests.py`

### 2.2 后端 LLM 调度连通性测试

这个脚本用于验证远程学校 API 和本地 Ollama 的调度逻辑是否正常：

```powershell
python test/orchestrator.py
```

通过标准：

- 输出中能看到“学校远程 API 优先调度”。
- 输出中能看到“Ollama 备选调度”。
- 说明调度链路本身可跑通。

### 2.3 前端 Supabase 连通测试

在前端工程目录运行：

```powershell
Set-Location frontend/frontend_vue
node .\test_modules\test-supabase-connection.js
```

通过标准：

- 能访问 `users` 表。
- 输出中包含“连接成功”。

### 2.4 前端 OSS 连通测试

在前端工程目录运行：

```powershell
node .\test_modules\test-oss-connection.js
```

通过标准：

- 能完成上传、下载、删除。
- 输出中包含“OSS 连接测试成功”。

### 2.5 前端远程模型连通测试

在前端工程目录运行：

```powershell
node .\test_modules\test-model-connection.js --base-url "https://chat.ecnu.edu.cn/open/api/v1/chat/completions" --api-key "你的学校 API Key" --model "ecnu-max"
```

通过标准：

- `models` 接口可访问。
- `chat.completions` 能返回模型回复。

### 2.6 后端接口最小连通测试

后端启动后可直接访问这些接口：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
Invoke-RestMethod http://127.0.0.1:8000/api/v1/health
```

通过标准：

- 返回 `{"status":"ok"}`。

---

## 3. 第二阶段：真正启动后端和前端服务

这一阶段开始测试真实服务。

### 3.1 启动后端服务

在仓库根目录打开一个新的 PowerShell 窗口，执行：

```powershell
conda activate charpick
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

启动后可先验证后端是否在线：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health
Invoke-RestMethod http://127.0.0.1:8000/api/v1/health
```

如果你要确认聊天接口可用，可以发一个最小请求：

```powershell
$body = @{
  message = "请用一句话介绍你自己"
  temperature = 0.1
  top_p = 0.9
  max_tokens = 128
} | ConvertTo-Json

Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:8000/chat `
  -ContentType 'application/json' `
  -Body $body
```

通过标准：

- `/health` 和 `/api/v1/health` 正常返回。
- `/chat` 返回 `response` 字段。

### 3.2 启动前端服务

在另一个 PowerShell 窗口进入前端目录：

```powershell
Set-Location E:\codes\CharPick\frontend\frontend_vue
npm install
npm run dev
```

启动后，Vite 默认地址通常是：

```text
http://127.0.0.1:5173
```

通过标准：

- 浏览器能打开前端首页。
- 控制台没有明显报错。
- 前端页面可以加载登录页、上传页、看板页等。

---

## 4. 第三阶段：真实前端页面测试

这一阶段是最重要的端到端测试，必须使用真实页面操作，而不是只看脚本输出。

### 4.1 登录页连通测试

打开浏览器访问：

```text
http://127.0.0.1:5173/login
```

检查点：

- 页面能正常渲染。
- 能完成登录或恢复会话。
- 登录后能够进入其它受保护页面。

### 4.2 上传页文件传输测试

打开：

```text
http://127.0.0.1:5173/upload
```

在页面上执行以下动作：

1. 选择一个真实文件，建议先使用 `data/test.txt`。
2. 填写书名（可选）。
3. 点击“开始上传”。
4. 观察是否成功上传到 OSS。
5. 观察上传后任务是否进入待解析或进行中状态。

检查点：

- 页面没有报错。
- 文件可以成功选择。
- 上传后任务列表能刷新。
- 卡片或任务里能看到对应的文件信息。

如果你想直接验证任务提取，可以在上传后点击“进行提取”。

### 4.3 看板页任务联通测试

打开：

```text
http://127.0.0.1:5173/dashboard
```

在页面里选择刚刚上传的书籍，然后检查：

- `chapter_extraction`、`summary`、`card` 任务统计是否出现。
- 任务列表是否有运行中、已完成、失败等状态。
- 是否能看到日志和逐章提取结果。
- summary 的本地路径和 OSS 路径按钮是否可点击。

检查点：

- `chapter_extractions` 中能看到本地文件和 OSS 文件。
- `summary` 目录能看到 `content_local_url` 与 `content_oss_url`。

### 4.4 角色卡页验证

打开：

```text
http://127.0.0.1:5173/characters
```

在页面里检查：

- 是否能筛选当前书籍。
- 是否能看到角色卡列表。
- 卡片中是否显示简介。
- 卡片的“本地目录”和“OSS 目录”按钮是否可点开。

如果你需要验证“石野”角色卡：

1. 先确认上传/提取流程里已经生成“石野”。
2. 在筛选或搜索框中输入“石野”。
3. 打开该卡片，确认标题、简介、更新时间和路径正常。

### 4.5 聊天页交互测试

打开：

```text
http://127.0.0.1:5173/chat
```

建议测试语句：

- `请用一句话介绍石野。`
- `根据当前书籍中的角色卡，简要概括石野的性格。`

检查点：

- 页面能发送消息。
- 后端能返回响应。
- 聊天结果能展示出来。

### 4.6 设置页联通测试

打开：

```text
http://127.0.0.1:5173/settings
```

检查并确认：

- `backend_url` 指向 `http://127.0.0.1:8000`
- 其它模型、API、OSS 配置与当前环境一致

如果设置页能正确保存配置，说明前端本地状态与后端调用配置链路都可用。

## 5. 推荐的一次完整执行顺序

你可以按下面的顺序实际执行：

### 第一步：跑自动化脚本

```powershell
python -m pytest -q backend/test/test_llm_dispatcher.py backend/test/test_llm_remote_client.py
python test/orchestrator.py
```

然后在前端目录运行：

```powershell
Set-Location frontend/frontend_vue
node .\test_modules\test-supabase-connection.js
node .\test_modules\test-oss-connection.js
node .\test_modules\test-model-connection.js --base-url "https://chat.ecnu.edu.cn/open/api/v1/chat/completions" --api-key "你的学校 API Key" --model "ecnu-max"
```

### 第二步：启动后端

```powershell
conda activate charpick
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

### 第三步：启动前端

```powershell
Set-Location E:\codes\CharPick\frontend\frontend_vue
npm run dev
```

### 第四步：打开真实页面验证

按顺序打开：

1. `/login`
2. `/upload`
3. `/dashboard`
4. `/characters`
5. `/chat`
6. `/settings`

### 第五步：确认端到端结果

至少确认这些结果：

- 后端 `/health` 正常。
- 前端页面能正常加载。
- 上传的文件能进入 OSS。
- 提取任务能进入后端并返回结果。
- 看板页能看到任务和日志。
- 角色卡页能看到卡片，并能打开本地和 OSS 路径。
- 聊天页能和后端交互成功。

## 6. 判定标准

### 后端连通通过

- `http://127.0.0.1:8000/health` 返回 `ok`。
- `http://127.0.0.1:8000/api/v1/health` 返回 `ok`。
- `/chat` 能返回模型回复。

### 前端连通通过

- `http://127.0.0.1:5173/login` 能打开。
- 页面路由能正常切换。
- Settings 页能读写 `backend_url`。

### 前后端连接通过

- 上传页能触发上传和提取。
- 看板页能查询到任务状态。
- 角色卡页能查到卡片和路径。
- 聊天页能从后端拿到回复。

## 8. 备注

- 如果你当前用的是远程学校模型，记得先确认 `backend/config.json` 中 `llm.remote_api` 的 `base_url`、`api_key` 和 `model_name` 可用。
- 如果前端上传页按钮可点但提取失败，优先检查后端 `/api/v1/extract` 和数据库连接。
- 如果角色卡页没有显示 `content_oss_url`，先确认远程上传已经完成。
