**数据库 传输 测试**

**目的**: 验证远程持久化模块在配置正确时能与数据库交互。

如何运行:
- 该模块需要环境变量 `SUPABASE_DB_URL` 或 `DATABASE_URL` 配置。将其写入仓库根目录下的 `.env` 或系统环境。
- 运行测试: `pytest test/db_tests.py -q`

说明:
- 如果未配置数据库，测试会被跳过。
- 若配置了真实数据库，测试将简单校验 `_resolve_db_url()` 是否返回有效 URL。

注意: 更深入的写入/读取测试需要一个可用的 Postgres 数据库和相应的表结构，当前测试仅做轻量探测，避免在 CI 中产生副作用。
