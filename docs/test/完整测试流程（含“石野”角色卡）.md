# 使用 data/test.txt 的完整测试流程（含“石野”角色卡）

本文档给出一套可直接执行的本地完整测试流程：
- 输入文件：`data/test.txt`
- 目标：跑通 L0 -> L2 全流程
- 结果：生成并定位“石野”角色卡

## 1. 前置准备

在项目根目录 `E:\codes\CharPick` 打开 PowerShell，执行：
```bash
conda activate charpick
```

```powershell
pip install -r requirements.txt
```

说明：
- 当前项目默认读取 `backend/config.json` 中的 LLM 配置。
- 若使用本地 Ollama，请将 `provider` 切到 `ollama` 并确保模型可用。
- 若使用远程接口，请确认 `remote_api` 的 `base_url`、`api_key`、`model_name` 可用。

## 2. 运行完整流程（L0 -> L2 + summary + card 本地全流程测试）
在项目根目录执行下面一条命令：
```powershell
python -c "from backend.workflow_runner import run_l0_to_l2_pipeline; import json; r = run_l0_to_l2_pipeline(source_path='data/test.txt', source_type='txt', source_file_id='sf_test_txt_001', file_name='test.txt', filter_noise=True, upload_chapters_to_remote=False, run_summary=True, run_card=True, card_character_name='石野', show_progress=True); print(json.dumps({'source_file_row': r.get('source_file_row'), 'chapter_count': len(r.get('chapter_rows', [])), 'l2_count': len(r.get('chapter_extraction_rows', [])), 'summary': r.get('summary'), 'card': r.get('card')}, ensure_ascii=False, indent=2))"
```

这条命令会完成：
- L0：原文读取与统一化
- L1：章节切分
- L2：逐章结构化提取（extraction_json）
- Summary：全书维度汇总
- Card：角色卡生成（会先从 `summary/characters/all_characters.json` 里选出“石野”对应条目，再结合章节证据，最终输出单个 `card.md` 文件）

## 3. 关键输出位置

运行完成后，输出会在：

```text
local_cache/test_sf_test_txt_001/
```

主要目录说明：
- `local_cache/test_sf_test_txt_001/unified_format/`：统一文本
- `local_cache/test_sf_test_txt_001/chapter_content_text/`：章节文本
- `local_cache/test_sf_test_txt_001/extraction_json/`：逐章提取 JSON
- `local_cache/test_sf_test_txt_001/summary/`：汇总结果，其中 `characters` / `items` 为 JSON，`storyline_events` / `world_locations` 为 Markdown
- `local_cache/test_sf_test_txt_001/card/`：角色卡结果，当前直接输出单个 Markdown 文件

## 4. 定位“石野”角色卡文件

角色卡由 `dispatch_card_prompts` 生成，文件命名规则：

```text
local_cache/test_sf_test_txt_001/card/sf_test_txt_001_石野_character_card.md
```

可用下面命令快速查看：

```powershell
Get-Content local_cache/test_sf_test_txt_001/card/sf_test_txt_001_石野_character_card.md -Encoding UTF8
```

## 5. 常见问题排查

1. 报错 `No chapter json files found`
- 原因：L2 未生成 `extraction_json`。
- 处理：先检查 LLM 配置是否可用，再重跑第 2 步。

2. 角色卡为空或缺字段
- 原因：输入文本中角色命中不足，或模型输出不稳定。
- 处理：确认 `data/test.txt` 含“石野”相关内容，必要时重跑并提高上下文质量。

3. 编码乱码
- 说明：流程中对 `txt/md` 会做 UTF-8 归一化，仍异常时请确认原始文件可读。

如果打印出 `.../card.md` 路径，即表示“石野”角色卡已产出。

## 7. 本地文件上传到远程服务器的测试（快速示例）

说明：上传到远程通常有两种途径——（A）通过 `run_l0_to_l2_pipeline(..., upload_chapters_to_remote=True)` 的函数参数直接在代码里触发远程入库；（B）先在本地产出 L2 JSON，再用远程 persistence 脚本或 API 上传。

A. 直接在流程里启用远程上传（示例，需在环境变量或 `config.json` 中配置数据库/OSS 信息）：

```powershell
# 设置环境变量（PowerShell 示例）
$env:SUPABASE_DB_URL = "postgresql://user:pass@db.example.com:5432/charpick"
$env:OSS_ENDPOINT = "https://oss.example.com"
$env:OSS_BUCKET = "mybucket"

python -c "from backend.workflow_runner import run_l0_to_l2_pipeline; import json; r = run_l0_to_l2_pipeline(source_path='data/test.txt', source_type='txt', source_file_id='sf_test_remote_001', file_name='test.txt', filter_noise=True, upload_chapters_to_remote=True, upload_summary_to_remote=True, upload_card_to_remote=True, card_character_name='石野', user_id='u_local_runner', username='local_runner', book_id='b_local_001', book_title='测试书'); print(json.dumps({'remote': r.get('remote_persist')}, ensure_ascii=False, indent=2))"
```

B. 先产出本地 L2，再调用上传脚本（更可控）：

```powershell
# 1) 仅生成本地 L2
python -c "from backend.workflow_runner import run_l0_to_l2_pipeline; r = run_l0_to_l2_pipeline(source_path='data/test.txt', source_type='txt', source_file_id='sf_test_local_l2', file_name='test.txt', upload_chapters_to_remote=False, run_summary=False, run_card=False); print(r['chapter_extraction_cache_dir'])"

# 2) 使用远程持久化脚本（如有）或手写脚本读取 local_cache 中的 JSON 并上传到 Supabase/OSS。
```

提示：若远程上传失败，请检查 `/memories/repo/supabase-connectivity.md` 中的网络兜底策略（pooler），并确认 `SUPABASE_POOLER_DB_URL` 等变量是否需要设置。

## 8. 本地后端运行 + 通过 API 触发并上传到服务器（生产流程模拟）

步骤概览：
- 启动后端 FastAPI；
- 使用 API `/start-extraction` 提交任务（可触发后台异步流程并写本地/远程）；
- 监控日志并检查本地/远程结果。

示例命令：

```powershell
# 1) 启动后端（在独立终端里运行）
conda activate charpick
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000

# 2) 在另一个终端，设置远程数据库环境并通过 HTTP API 发起提取任务
$env:SUPABASE_DB_URL = "postgresql://user:pass@db.example.com:5432/charpick"

Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8000/start-extraction -Body (ConvertTo-Json @{file_name='test.txt'; prompt='提取角色卡：石野'; filter_noise=$true}) -ContentType 'application/json'

# 3) 监控实时日志流（SSE）
# 在浏览器打开或用工具读取： http://127.0.0.1:8000/logs

# 4) 提交完成后，若远端上传开启，可在 Supabase 或目标数据库查看相应表（示例表：chapter_extractions / card / summary）
```

常见注意事项：
- 若 API 返回 404 表示 `data/test.txt` 不存在，请确保服务运行目录为项目根并且 `data/test.txt` 可访问；
- 若远程上传超时或网络错误，建议先把 `upload_chapters_to_remote=False` 保持本地产物，再运行单独的远程上传工具以便重试和日志跟踪。
