# 前后端对接改造记录

## 本次改造目标
- 将后端封装为统一可调度端口（FastAPI）。
- 前端通过统一端口触发提取调度。
- 提取结果由后端上传远程数据库/OSS，前端从远程库读取展示。
- 保留旧接口兼容（/files、/start-extraction、/logs）。

## 已完成操作

### 1. 后端统一调度接口重构
文件：backend/main.py

完成内容：
- 新增统一健康接口：
  - GET /health
  - GET /api/v1/health
- 新增提取调度接口：
  - POST /api/v1/extract
- 新增调度任务状态接口：
  - GET /api/v1/tasks/{task_id}
- 新增角色查询接口：
  - GET /api/v1/characters
  - GET /api/v1/characters/{character_id}
- 新增聊天代理接口：
  - POST /chat
- 保留旧接口兼容：
  - GET /files
  - GET /logs
  - POST /start-extraction

### 2. 后端调度流程接入远程入库
文件：backend/main.py（调用 backend/workflow_runner.py）

接口 /api/v1/extract 已接入：
- run_l0_to_l2_pipeline(...)
- upload_chapters_to_remote=True
- upload_summary_to_remote=True（可请求覆盖）
- upload_card_to_remote=True（可请求覆盖）

效果：
- L0/L1/L2、summary、card 的文件会被上传并写入远程数据库记录。
- 前端通过 Supabase 读取 books/extraction_tasks/chapter_extractions/summary/card 表即可看到结果。

### 3. 前端对接统一端口

#### 3.1 默认后端地址
文件：frontend/frontend_vue/src/stores/settings.js

改动：
- backend_url 默认值从空字符串改为 http://127.0.0.1:8000

目的：
- 前端开箱即可访问统一后端端口，不需要每次手工配置。

#### 3.2 上传页提取触发逻辑
文件：frontend/frontend_vue/src/pages/UploadPage.vue

改动：
- 修复“无角色不允许提取”的阻塞逻辑。
- 当 getBookRoles 为空时也可触发 /api/v1/extract。
- 自动传入 card_character_name（优先取角色列表第一个 name，否则默认石野）。

目的：
- 避免“必须先有角色才能提取角色”的循环依赖。

### 4. 远程数据库与前端对接说明

已存在并继续复用的前端读取链路：
- extraction_tasks：任务状态
- chapter_extractions：L2 聚合结果
- summary：汇总结果
- card：角色卡结果

这些读取逻辑在 frontend/frontend_vue/src/api/modules/*.js 中已实现，无需迁移。

## 调用示例

### 1) 触发统一调度
POST /api/v1/extract

请求头：
- Authorization: Bearer <auth_token>

请求体示例：
```json
{
  "book_id": "book_xxxx",
  "roles": ["char_001"],
  "is_dynamic": false,
  "card_character_name": "石野",
  "run_summary": true,
  "run_card": true,
  "upload_summary_to_remote": true,
  "upload_card_to_remote": true
}
```

返回示例：
```json
{
  "task_id": "dispatch_ab12cd34ef56",
  "book_id": "book_xxxx",
  "status": "started",
  "message": "提取任务已启动，角色卡目标：石野",
  "source_file_id": "sf_book_xxxx_1710000000"
}
```

### 2) 查询调度状态
GET /api/v1/tasks/{task_id}

返回示例：
```json
{
  "task_id": "dispatch_ab12cd34ef56",
  "status": "running",
  "progress": 74,
  "stage": "角色卡生成",
  "message": "开始生成角色卡：石野"
}
```

## 运行建议

后端启动：
```powershell
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

前端启动：
```powershell
cd frontend/frontend_vue
npm install
npm run dev
```

## 兼容性说明
- 旧前端若还调用 /start-extraction，不会立刻失效。
- 新前端建议统一走 /api/v1/extract。
- 聊天页仍走 /chat（或直接走 OpenAI 兼容地址）。
