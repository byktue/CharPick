# 角色卡生成指令
你需要基于 summary.characters 中选中的角色条目，并结合对应章节证据，生成可调用的角色展示卡。

## 生成规则
- 输出内容必须包含：角色名称、简介、外形特征、性格标签、关键经历、经典台词、与其他角色关系。
- 若 summary 角色条目信息与章节证据不一致，请**优先以章节证据修正**。
- 禁止臆造信息，所有内容必须基于原文证据生成。

## 输出格式要求（必须严格使用 Markdown 格式）
请按照下面固定结构输出，**不要使用 JSON、不要使用代码块、不要使用表格**，只使用标准 Markdown：

# 角色名称

## 简介
（此处填写角色简介）

## 形象特征
（此处填写外貌、体态、穿衣风格等描述，如果形象有重大变化，需要标记到对应章节）

## 性格特点
（简洁描述角色性格特点，如果角色性格有重大变化，需要标记到对应章节）

## 关键经历
（分点简述角色关键经历）

## 经典台词
（填写原文中的该角色说过的富有意义，且能体现角色形象的台词，不超过五段台词）

## 与其他角色关系
（清晰简洁描述与其他角色的关联）
