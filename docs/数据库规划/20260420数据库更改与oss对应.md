## 数据库部分
### user表

| 字段名 | 数据类型 | 必要性 | 介绍 |
| :--- | :--- | :--- | :--- |
| user_id | text | 必要 | 用户的唯一标识符，格式为`user_XXXXX`。 |
| username | text | 必要 | 用户登录或显示的昵称，通常不可重复。 |
| email | text | 非必须 | 用户的电子邮箱地址，常用于登录、找回密码或接收通知。 |
| password_hash | text | 必要 | 用户密码的加密哈希值，出于安全考虑，不存储明文密码。 |
| status | text | 非必须 | 用户当前的状态（例如：`active` 激活、`banned` 封禁、`pending` 待审核）。 |
| created_at | timestamptz | 必要 | 记录该用户账号创建的时间（带时区的时间戳）。 |
| updated_at | timestamptz | 必要 | 记录该用户信息最后一次更新的时间，用于数据同步或审计。 |
| avatar | text | 非必须 | 用户头像的oss链接地址。 |
| prompt | text | 非必须 | 用户的自定义提示词或系统指令（常见于 AI 类应用，用于设定用户偏好或角色）。 |


### books表

| 字段名           | 数据类型        | 必要性 | 介绍                                                                 |
| :------------ | :---------- | :-- | :----------------------------------------------------------------- |
| book_id       | text        | 必要  | 书籍的唯一标识符，格式为`book_XXXXX`。                                          |
| user_id       | text        | 必要  | 关联的用户 ID（外键），表明这本书属于哪个用户。                                          |
| title         | text        | 必要  | 书籍的标题名称。                                                           |
| author        | text        | 非必须 | 书籍的作者姓名。                                                           |
| source_type   | text        | 必要  | 书籍的来源类型（例如：`txt`、`epub`、`pdf` 等）。                                  |
| language      | text        | 非必须 | 书籍的语言代码（例如：`zh-CN`, `en`）。                                         |
| status        | text        | 必要  | 书籍当前的任务状态（例如：`ready` 就绪、`processing` 处理中、`succeed`成功、`failed` 失败）。 |
| book_file_url | text        | 必要  | 书籍文件的oss存储地址链接。                                                    |
| created_at    | timestamptz | 必要  | 记录创建的时间（带时区的时间戳）。                                                  |
| updated_at    | timestamptz | 必要  | 记录最后一次更新的时间。                                                       |
| progress      | numeric     | 非必须 | 处理进度（例如：1-100代表百分比）。                                               |
| err_message   | text        | 非必须 | 如果处理过程中发生错误，存储具体的错误信息文本。                                           |

### summary表

| 字段名 | 数据类型 | 必要性 | 介绍 |
| :--- | :--- | :--- | :--- |
| summary_id | text | 必要 | 摘要的唯一标识符，格式为`book_XXXXX_summary_type`。 |
| book_id | text | 必要 | 关联的书籍 ID（外键），指明该摘要属于哪本书。 |
| type | text | 必要 | 摘要的类型（例如：`characters` 角色总表、`items` 物品总表、`storyline_events`剧情时间线、`world_locations`世界观/地点表）。 |
| name | text | 非必须 | 摘要的名称或标题，用于区分不同类型的总结。 |
| created_at | timestamp | 必要 | 记录创建的时间。 |
| updated_at | timestamp | 必要 | 记录最后一次更新的时间。 |
| prompt_id | text | 非必须 | 生成此摘要时所使用的提示词 ID，用于追溯生成逻辑。 |
| content_local_url | varchar | 非必须 | 摘要内容在本地服务器的存储路径或文件地址。 |
| content_oss_url | varchar | 必须 | 摘要内容在对象存储服务上的云端访问链接。 |

| :---------------- | :-------- | :-- | :------------------------- |

| type              | text      | 必要  | 卡片的类型                      |
| name              | text      | 必要  | 卡片的名称                      |

| updated_at        | timestamp | 必要  | 记录最后一次更新的时间。               |
| prompt_id         | text      | 非必须 | 生成此卡片内容时所使用的提示词 ID。        |
| content_local_url | text      | 非必须 | 卡片相关内容（如图片或附件）在本地服务器的存储路径。 |

## oss对应逻辑

```txt
oss/
└── {user_id}/                  # 用户根目录 (例如: user_12345/)
    ├── Avatar/                 # 头像目录
    │   └── avatar.png
    └── Books/                  # 书籍根目录
        └── {book_id}/          # 书籍目录 (例如: book_67890/)
            ├── source.txt      # 原始上传文件 (对应 books.book_file_url)
            ├── summary/        # 摘要文件目录
            │   └── {summary_id}/
            │       └── content.json (对应 summary.content_oss_url)
            └── card/           # 卡片文件目录
                └── {card_id}/
                    └── content.md   (对应 card.content_oss_url)

```

```
oss
 - user_id 根据用户user_id创建的目录
    - Avatar 创建Avatar目录 存储用户的头像图像
    - Books 创建Books目录 存储用户上传的书籍
        -book_id 根据书本book_id创建的目录
            - txt 上传的书本
            - summary 会有summary_id，根据summary_id创建目录，存储指定的文件
            - card 会有card_id，根据card_id创建目录，存储指定的文件
```
