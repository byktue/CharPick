# CharPick 数据库表结构汇总（**带精简注释版**）
所有表统一放在 **单个 PostgreSQL 库**，通过 **user_id 做数据隔离**，直接对接前端接口。

| 层级       | 对应业务表                                          | 这层存什么                                                     |
| -------- | ---------------------------------------------- | --------------------------------------------------------- |
| L0 原始层   | books、source_files                             | 书籍主入口、原始文件、导入元数据、OCR 来源                                   |
| L1 章节层   | chapters（本地缓存）                                 | 切分后的章节文本                                                  |
| L2 提取层   | chapter_extractions                            | 每章抽出的角色、物品、剧情、地点、时间线 JSONB，本地缓存，且传输到远程数据库（supabase + oss） |
| L3 汇总层   | summary                                        | 面向前端展示书籍汇总提取的结果（人物、物品、剧情、世界观）                             |
| L4 展示调度层 | card                                           | 面向前端展示角色库的最终业务结果                                          |
| 通用支撑     | users、auth_sessions、extraction_tasks、task_logs | 用户身份、登录态、任务过程、任务日志                                        |


---
## 一、用户与认证表
### 1. users（用户主表）
- id（PK）# 用户唯一ID，主键 M
- username # 用户名 M
- email # 邮箱
- password_hash # 密码哈希（不存明文） M
- status # 账号状态：正常/禁用
- created_at # 创建时间 M
- updated_at # 更新时间 M

### 2. auth_sessions（会话/Token 表）
- id（PK）# 会话唯一ID M
- user_id（FK → users）# 所属用户（外键） M
- access_token_jti # 访问令牌唯一标识 M
- refresh_token_hash # 刷新令牌哈希 M
- expires_at # 过期时间 M
- revoked_at # 撤销时间（NULL=有效）
- created_at # 登录时间 M
- updated_at # 最后更新时间 M

---

## 二、书籍与原始文件表
### 3. books（书籍/项目主表）
- id（PK）# 书籍唯一ID M
- user_id（FK → users）# 所属用户（外键） M
- title # 书名/项目名 M
- author # 原著作者
- source_type # 来源类型：epub/txt/pdf/image M
- language # 语言：zh-CN/en
- status # 处理状态：待解析/解析中/已完成/失败 M
- cover_url # 封面图片地址
- created_at # 创建时间 M
- updated_at # 更新时间 M

### 4. source_files（源文件表 L0）
- id（PK）# 源文件ID M
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- file_name # 原始文件名 M
- file_url # 文件存储地址 M
- file_hash # 文件哈希（防重复） M
- mime_type # 文件类型
- unified_format_url # 中间结构地址（实际文件存储在本地缓存，本地缓存用txt格式存储）
- parse_status # 解析状态 M
- created_at # 创建时间 M
- updated_at # 更新时间 M
- chapter_prepare_url # chapters表切换到这里，然后chapters表整表缓存于本地 N

---

## 三、章节与提取表（L1/L2）
### 6. chapter_extractions（章节提取表 L2）(按书籍+提取类型合并存储)
- id（PK）# 提取记录ID M
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- extractor_type VARCHAR(20) # 提取类型（character/plot/item等）M
- book_extraction_json_local_url（TEXT）# 整书同类型提取结果的本地文件地址 M
- book_extraction_json_oss_url（TEXT）# 整书同类型提取结果的OSS远程存储的文件地址 M

- created_at # 创建时间 M
- updated_at # 更新时间 M
- embedding_vector（pgvector）# 向量数据
- prompt_id （FK → prompt_config）# 提示词id
- model_name # 使用模型

补充extractor_type解释

| 常见 `extractor_type` 枚举值 | 对应提取任务                       |
| ----------------------- | ---------------------------- |
| `character`             | 角色提取（提取本章出现的角色、人设）           |
| `plot`                  | 剧情提取（提取本章的剧情事件、时间线）          |
| `item`                  | 物品提取（提取本章出现的道具、装备）           |
| `location`              | 地点提取（提取本章出现的场景、地点）           |
| `summary`               | 章节摘要提取（生成本章的总结）              |
| `full`                  | 全量提取（一次性提取角色 + 剧情 + 物品等所有内容） |

## 四、汇总结果表（L3 前端核心）

### summary（总表模板）
- id (PK) # 表唯一 ID M
- user_id (FK → users) # 所属用户 M
- book_id (FK → books) # 所属书籍 M
- type # 表类型 M
- name # 表名称
- content_local_url # 该表详细内容本地缓存地址
- content_oss_url # 该表详细内容地址，具体文件存储于oss中
- created_at # 创建时间 M
- updated_at # 更新时间 M
- prompt_id （FK → prompt_config）# 提示词id


| type             | name    |
| ---------------- | ------- |
| characters       | 角色总表    |
| items            | 物品总表    |
| storyline_events | 剧情时间线   |
| world_locations  | 世界观/地点表 |

### card （展示卡模板）
- id (PK) # 唯一ID M
- user_id (FK → users) # 所属用户 M
- book_id (FK → books) # 所属书籍 M
- type # 类型 M
- name # 名称 M
- intro # 简介
- content_local_url # 该卡详细内容地址，具体文件存储于本地缓存
- content_oss_url # 该卡详细内容地址，具体文件存储于oss中
- created_at # 创建时间 M
- updated_at # 更新时间 M
- prompt_id （FK → prompt_config）# 提示词id

### prompt_config （设置版块）
- id (PK) # 唯一ID M
- user_id (FK → users) # 所属用户 M
- type # 类型 M
- prompt_url # prompt详细内容地址，具体文件存储于oss中 M
- created_at # 创建时间 M
- updated_at # 更新时间 M

type分类
- 逐章提取 chapter_prompts
- 总表提取 summary_prompts
- 卡提取 card_prompts

---

## 五、媒体资源表
### 12. media_assets（图片/封面/角色立绘）
- id（PK）# 媒体资源ID
- user_id（FK → users）# 所属用户
- book_id（FK → books）# 所属书籍
- asset_type # 资源类型：封面/头像/表情包
- object_url # 原图地址
- thumbnail_url # 缩略图地址
- tags_json（JSONB）# 标签
- embedding_vector（pgvector）# 图片向量
- source_character_id（FK → characters）# 关联角色

---

## 六、任务与日志表
### 13. extraction_tasks（异步任务表）
- id（PK）# 任务ID M 
- user_id（FK → users）# 所属用户 M
- book_id（FK → books）# 所属书籍 M
- task_type # 任务类型：upload/chapter_extraction/summary/card M
- status # 任务状态 M
- progress # 进度百分比 M
- error_message # 错误信息
- created_at # 创建时间 M
- updated_at # 更新时间 M

### 14. task_logs（任务日志）
- id（PK）# 日志ID
- task_id（FK → extraction_tasks）# 所属任务
- user_id（FK → users）# 所属用户
- level # 日志级别
- message # 日志信息
- detail_json（JSONB）# 详细数据
- created_at # 记录时间

---

## 极简对应关系
- **用户** → users / auth_sessions
- **一本书** → books + source_files
- **章节解析** → chapters
- **章节提取** → chapter_extractions
- **前端展示核心** → characters / character_versions / items / storyline_events / world_locations
- **图片资源** → media_assets
- **异步任务** → extraction_tasks / task_logs
