# CharPick 数据库表结构汇总（精简可直接用）
所有表**统一放在单个 PostgreSQL 库**，全程通过 **user_id 做数据隔离**，业务关系清晰、接口直接对应。

---

## 一、用户与认证表
### 1. users（用户主表）
- id（PK）
- username
- email
- password_hash
- status
- created_at
- updated_at
### 2. auth_sessions（会话/Token 表）
- id（PK）
- user_id（FK → users）
- access_token_jti
- refresh_token_hash
- expires_at
- revoked_at
- created_at
- updated_at

---

## 二、书籍与原始文件表
### 3. books（书籍/项目主表）
- id（PK）
- user_id（FK → users）
- title
- author
- source_type
- language
- status
- cover_url
- created_at
- updated_at
### 4. source_files（源文件表 L0）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- file_name
- file_url
- file_hash
- mime_type
- raw_text_url
- ocr_text_url
- parse_status

---
## 三、章节与提取表（L1/L2）
### 5. chapters（章节表 L1）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- chapter_no
- chapter_title
- chapter_range
- story_time
- content_text
- word_count
- source_file_id（FK → source_files）
### 6. chapter_extractions（章节提取表 L2）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- chapter_id（FK → chapters）
- extractor_type
- extraction_json（JSONB）
- embedding_vector（pgvector）
- prompt_version
- model_name
- confidence

---
## 四、汇总结果表（L3 前端核心）
### 7. characters（角色总表）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- name
- alias
- role_type
- first_appearance_chapter_id
- static_profile_json（JSONB）
- dynamic_profile_json（JSONB）
- prompt_text
- avatar_url
- status
### 8. character_versions（角色动态版本表）
- id（PK）
- user_id（FK → users）
- character_id（FK → characters）
- book_id（FK → books）
- version_no
- chapter_range
- profile_json（JSONB）
- prompt_text
- is_default
### 9. items（物品表）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- name
- description
- owner_character_id（FK → characters）
- first_appearance_chapter_id
- item_json（JSONB）
### 10. storyline_events（剧情时间线）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- chapter_id（FK → chapters）
- event_title
- event_summary
- participants_json（JSONB）
- location_id
- event_time
### 11. world_locations（世界观/地点表）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- name
- description
- related_chapter_ids（JSONB）
- location_type

---

## 五、媒体资源表
### 12. media_assets（图片/封面/角色立绘）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- asset_type（cover/avatar/emoji等）
- object_url
- thumbnail_url
- tags_json（JSONB）
- embedding_vector（pgvector）
- source_character_id（FK → characters）

---

## 六、任务与日志表
### 13. extraction_tasks（异步任务表）
- id（PK）
- user_id（FK → users）
- book_id（FK → books）
- task_type（upload/parse/extract/summary）
- status
- progress
- error_message
- created_at
- updated_at
### 14. task_logs（任务日志）
- id（PK）
- task_id（FK → extraction_tasks）
- user_id（FK → users）
- level
- message
- detail_json（JSONB）
- created_at

---

# 极简对应关系（一眼看懂）
- **用户** → users / auth_sessions
- **一本书** → books + source_files
- **章节解析** → chapters
- **章节提取** → chapter_extractions
- **前端展示核心** → characters / character_versions / items / storyline_events / world_locations
- **图片资源** → media_assets
- **异步任务** → extraction_tasks / task_logs
