# 详细
### 🏛️ 成员 A：后端引擎与数据中台（逻辑核心）

**目标**：确保输入一个文件，能产出结构化、可调用的角色知识库。
#### **1. 万能解析与预处理（L0 -> L1）**
- **开发解析算子**：编写针对 `epub` (ebooklib)、`pdf` (PyMuPDF)、`txt` 的解析脚本。
- **物理/逻辑分块**：实现按“正文章节”切分文本，并为每一块打上 `chapter_id` 和 `word_count` 标签。
- **数据库落地**：在 PostgreSQL 中建立 `chapters` 表，存储 L1 层的纯净文本

#### **2. 提取工作流实现（L1 -> L2）**
- **LLM 推理环境**：部署 **vLLM/Ollama**，确保 API 端口可用
- **Map 任务开发**：编写 Prompt，让 LLM 针对单章文本提取 JSON（包含：本章登场角色、发生的关键事件、提到的地点）
- **异步调度**：利用 Python `asyncio` 并行调用 LLM，将结果存入 **L2 细分特征层**

#### **3. 全局汇总算法（L2 -> L3）**
- **Reduce 任务开发**：编写聚合逻辑，将 L2 的碎片化信息（如：第 1、5、10 章都提到了“石野”）融合成 **L3 全局画像**
- **产出物**：生成标准角色 JSON（姓名、性格、全局经历汇总）及剧情时间线列表

### 🎨 成员 B：交互前端与数据调度（应用外壳）
**目标**：让用户能方便地上传、查看，并与提取出来的角色说话。
#### **1. 简易交互前端开发（PC & 移动端适配）**
- **技术选型**：使用 **Streamlit**。它天然适配 PC 和移动端浏览器，开发速度最快。
- **功能模块**：
    - **上传页**：支持 `st.file_uploader` 接收多格式文件，并触发后端解析。
    - **进度条**：调用后端状态，展示“正在解析章节...正在提取人设...”的进度。
    - **看板页**：展示 L3 层的数据（角色卡片流、剧情折叠时间线、世界观概要）。

#### **2. 角色调度与交互中心**
- **酒馆 AI 对接**：编写转换脚本，将 L3 层的角色 JSON 动态转化为 **SillyTavern** 兼容的角色卡。
- **简易对话框**：在 Streamlit 中集成一个 `st.chat_message` 对话框，用户点击某个角色，前端自动加载该角色的 Persona 作为 System Prompt，调用后端 LLM 进行对话。

#### **3. 自动化来源整理（辅助功能）**
- **初级爬虫**：编写一个简单的脚本，支持输入网址或本地路径，自动完成文件的收集、命名对齐（元数据：标题、作者、类型）并推入待处理队列。
