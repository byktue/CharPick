# 提交到远程数据库操作指南（Supabase）

本文用于将本地 schema 提交到 Supabase 远程 PostgreSQL。
当前一期 schema 文件：
- `database/001_init_auth_l0_l2.sql`

包含表：
- users
- auth_sessions
- books
- source_files
- chapters
- chapter_extractions

---

## 方式 A：Supabase Dashboard SQL Editor（推荐）

适用场景：
- 想最直观地执行 SQL
- 不想在本地配置数据库连接串

步骤：
1. 打开 Supabase 项目控制台。
2. 进入 SQL Editor。
3. 新建 Query。
4. 粘贴 `database/001_init_auth_l0_l2.sql` 全部内容。
5. 点击 Run 执行。
6. 在 Table Editor 验证上述 6 张表是否创建成功。

优点：
- 操作简单
- 可视化查看执行结果

---

## 方式 B：本地脚本提交到 Supabase（推荐用于重复执行）

适用场景：
- 需要反复执行建表流程
- 需要纳入团队操作流程

### 1) 安装依赖

```bash
pip install supabase python-dotenv psycopg2-binary
```

### 2) 配置环境变量

在项目根目录创建 `.env`（若已存在则补充）：

```env
SUPABASE_DB_URL=postgresql://postgres:<db_password>@db.<project-ref>.supabase.co:5432/postgres?sslmode=require
```

其中：
- `<db_password>`：Supabase 数据库密码
- `<project-ref>`：Supabase Project Ref（不是 API URL 全串）

### 3) 执行脚本

```bash
python database/bootstrap.py
```

成功后会输出：
- Schema applied successfully: auth + L0 + L1 + L2

## 方式 C：psql 直连执行（可选）

适用场景：
- 本地已安装 psql
- 习惯 SQL 命令行

```bash
psql "host=db.<project-ref>.supabase.co port=5432 dbname=postgres user=postgres password=<db_password> sslmode=require" -f database/001_init_auth_l0_l2.sql
```

## 提交前检查清单

1. SQL 文件版本正确（本期为 001_init_auth_l0_l2.sql）。
2. 目标项目正确（确认 Project Ref）。
3. 使用 SSL 连接（sslmode=require）。
4. 执行后确认 6 张表与索引存在。

---

## 常见问题

### 1) 报错：type "vector" does not exist
原因：未启用 pgvector 扩展。
处理：确认 SQL 顶部已执行：
- create extension if not exists vector;

### 2) 报错：permission denied for schema public
原因：当前数据库角色权限不足。
处理：使用 Supabase 项目默认 postgres 用户执行，或在 SQL Editor 中使用有权限身份执行。

### 3) 重复执行报错 trigger already exists
原因：触发器未用 if not exists 保护。
处理：首次执行应正常。若需幂等化，可改为“先 drop trigger if exists 再 create trigger”。

## 建议的团队流程

1. 每次 schema 变更新增一个 SQL 文件，例如：
- 002_add_xxx.sql
- 003_add_xxx.sql
2. 远程执行后记录执行时间和执行人。
3. 生产环境执行前先在测试项目验证。
